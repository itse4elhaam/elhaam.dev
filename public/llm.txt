## Analytics & Feature Flags

**Analytics Provider:** PostHog
- **Events Tracked:** Pageviews, article reads, text shares, custom events
- **Session Recordings:** Enabled for user behavior analysis
- **Data Retention:** Configurable (default 90 days)
- **Privacy:** GDPR compliant, can be anonymized

**Feature Flags Implemented:**
- `new-homepage-design` - A/B test new homepage layouts
- `author-cta` - Show/hide "Work with me" CTA on blog posts

**Feature Flag SDKs Used:**
- **Client-side:** posthog-js with React hooks (`useFeatureFlagEnabled`, `usePostHog`)
- **Server-side:** posthog-node with local evaluation

## PostHog Integration Details

### Client-side Setup (posthog-js)
- Provider: `PostHogClientProvider` wrapper component
- Config: API key, host, person_profiles, capture_pageview, capture_pageleave
- React hooks: `usePostHog`, `useFeatureFlagEnabled`, `useActiveFeatureFlags`

### Server-side Setup (posthog-node)
- Library: `posthog-node` with PostHog class
- Features: Local evaluation, polling interval 30s
- Methods: `getAllFlags`, `isFeatureEnabled`, `shutdown()`

### Events Tracked
1. `$pageview` - Automatic on page load
2. `text_shared` - When user shares selected text
3. `article_read` - When user views blog post

### Feature Flag Usage
```typescript
// Client-side
const isEnabled = await posthog.isFeatureEnabled("flag-key");

// Server-side  
const flags = await client.getAllFlags(distinctId);
```
